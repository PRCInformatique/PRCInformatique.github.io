Les Exceptions

Une exception, c'est une erreur levée par l'interprèteur python. Il existe plusieurs types d'exception propre a divers type d'erreur. On peut nous même en créer une.
Cependant, il faut comprendre que une fois que l'interprèteur python rencontre une exception, il intéromp totalement le programme. Par conséquent, Python nous donne la possibilité de gérer ces exceptions et de soit continuer le programme en les ignorants, soit effectuer une action. Pour ce fait, nous avons quatres mots clés python qui ont tous une utilisation particulière. soit try, except, else, finally. La synthase est:
try:
instruction de texter
except:
action a faire en cas d'exception
else:
instruction a faire si il n'y a pas d'erreur
finally:
instruction a faire si il y a erreur ou pas.
Pour une interprétation littéraire de cette synthase, on dira: Essaie l'instruction de texte. Si une erreur est identifié, au lieu de quiter le programme, tu excécutes l'instruction au niveau de except. Sinon, tu excécute l'instruction au niveau de else. A la fin du test, tu excécutes l'instruction de finally. . Cependant, pour un controle plus précis du type d'actions a faire en cas d'exception, il est conseiller de préciser le type d'exception au niveau de except. Cette ligne devient except type d'exception:
On peut aussi recuperer le type d'erreur en utilisant le mot clé as suivie d'une varriable de recuperation. Synthase: except type_erreur as variable:

Assertions

C'est un moyens de s'assurer du respect d'une condition précis. On utilise le mots clé assert suivie de la condition. Si cette condition n'est pas respection, un type d'erreur est levé au niveau de python et le programme s'arrète a moins qu'on definie un test de gestions d'erreur vus plustôt. Le type d'erreur qu'il lève est AssertionError.

Lever une exception personnelle

Bien sur, nous pouvons lever nos propres erreurs. C'est bien utile lorsque l'on veut faire respecter une condition propre a nous que l'interprèteur python ne considère pas comme erreur. pour le faire, il suffit d'utiliser le mot clé raise suivie du type et du message d'erreur souhaiter. Soit la Synthase: raise type_erreur ("message d'erreur").

En résumé:

  • On peut intercepter les erreurs grâce aux blocs try, except, else, finally
  • La synthase d'une assertion est assert test
  • Les assertions lèvent une exception AssertionError si le test échoue
  • On peut lever une exception grâce au mot-clé raise suivi du type et du message d'erreur
