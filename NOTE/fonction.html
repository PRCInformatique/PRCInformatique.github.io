Les fonctions à plusieurs paramètre:

En d'autre terme, on dis les fonctions dont le nombre de paramètre et même le type n'est pas connus d'avance. Nous pouvons prendre l'exemple de print(). Il prend un nombre infinie de paramètre et les affiches l'un après l'autre en les séparants par des espaces. Il peut prendre n'importe quel type d'objet en paramètre. Ce n'est pas sorcier. Voici comment il est définie:
print(*value,sep= ' ', end = '\n',file = sys.stdout). Dans ce code, le premier argument *value est la paramètre qui lui permet de prendre en compte toutes les types d'objets et ce peut import le nombre. Les autres paramètres sont des paramètres ditent nommés qui sont définis par la fonction par défaut. Cependant, on peut changer leur valeur en les nommants.
Ainsi, pour définir une telle fonction, la synthase est: def fonction (*parametre_multiple, paramètre_nommé). La suivie de l'ordre est très important et la paramètre_nommé est facultatif.
Nous pouvons avec cette méthode transformer des listes en un ensemble de paramètre pour une fonction deja définie. Suposons que nous avons une fonction qui prend en priorie deux arguments et que l'on à dans une liste ces deux arguments. Au lieu de les passer en passant par le indexing, on peut tout simplement passer en parametre la liste précédée d'un *. En passant par cette façons, on demande a python de décomposer la liste. Ainsi, une erreur est levé si le nombre de paramètres exiger par la fonction n'est strictement pas égale à la taille de la liste. Cette fonctionnalité est utile et indispensable pour certain actions dans le langage.
